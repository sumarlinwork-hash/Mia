function mapState(app):

  if app.updating:
    return "UPDATING"  // Special case

  if app.error:
    return "ERROR"  // Special case

  if app.running:
    return "ACTIVE"

  if app.installed:
    return "READY"

  return "AVAILABLE"

### B.3 CTA Logic Implementation

File: frontend/src/utils/getCTA.ts

function getCTA(app):

  state = mapState(app)

  IF state == "AVAILABLE":
    return ["Use", "Try"]

  IF state == "READY":
    return ["Open"]

  IF state == "ACTIVE":
    return ["Running"]  // disabled button

  IF state == "UPDATING":
    return ["Updating..."]  // disabled

  IF state == "ERROR":
    return ["Fix", "Retry"]  // error recovery

### Cost Control Implementation

PROBLEM:
LLM previews are expensive and slow at scale.

Solution: Multi-mode preview system.

Extend preview config in manifest:

"preview": {
  "type": "interactive",
  "mode": "static | template | light_llm",
  "template": "chatbot_demo"
}

Execution Rules:

IF mode == "static":
  → return predefined response
  → Cost: $0
  → Speed: < 100ms

IF mode == "template":
  → generate response using template logic (no LLM)
  → Cost: $0
  → Speed: < 500ms

IF mode == "light_llm":
  → call LLM with constraints
  → Cost: $0.01-0.02 per preview
  → Speed: < 1s
  → Apply rate limiting

Rate Limiting (for light_llm mode):

* max 5 requests per session per user
* timeout guard (3s max)
* fallback to template mode if LLM fails

Update Preview Engine:

File: backend/discovery/preview_engine.py

function run_preview(mode, template, input):

  if mode == "static":
    return static_response()

  if mode == "template":
    return template_response(template, input)

  if mode == "light_llm":
    check_rate_limit()
    return llm_response(input, constraints={
      "max_tokens": 100,
      "timeout": 3
    })

Recommended mode distribution:

* 70% static (simple demos)
* 20% template (interactive but rule-based)
* 10% light_llm (complex AI demos only)

Cost impact:

* 1000 previews in static mode = $0
* 1000 previews in light_llm mode = $10-20
* Multi-mode saves potentially thousands
